## 🎯 场景驱动的翻车模式认知框架

> **为什么三大场景要分开看**：同一种 VLA 失败，在不同场景的**容忍度、根因概率、兜底手段**完全不同

| # | 失败类型 | 场景A 仓储物流 | 场景B 制造业 | 场景C 农业 |
|:---:|:---|:---:|:---:|:---:|
| 1 | **新物体/配置** | 🟡 品种多，翻车频繁但可接受 | 🔴 偶尔换料，翻了就严重 | 🟢 环境稳定，很少新品种 |
| 2 | **遮挡/视角切换** | 🔴 货架密集，遮挡频繁 | 🟡 结构化环境可控 | 🟢 开阔空间 |
| 3 | **极端光照** | 🟡 仓库灯光变化 | 🟢 产线灯光稳定 | 🔴 室外光照变化巨大 |
| 4 | **高精度任务** | 🟢 精度要求低（2cm OK） | 🔴 0.5mm 精度是生命线 | 🟢 精度要求中等 |
| 5 | **动态环境** | 🔴 人+AMR+其他机器人共存 | 🟡 人需保持距离 | 🟢 动物会有不确定行为 |

### 场景关键洞察

> **场景A 的噩梦**：品种突然多了 50 种没训练过的 → 兜底策略：视觉重定向
> **场景B 的死穴**：精度从 99% 掉到 92% → 兜底策略：力控保护 + 停机检查
> **场景C 的麻烦**：早中晚光线完全不同 → 兜底策略：传感器冗余 + 多模态融合

## 第 15 天 - VLA 翻车模式清单（场景增强版）

### 学习重点

理解 VLA 失败的 **5 大类场景**：

1. **新物体/配置** — 训练数据没见过
   - 例：系统在快递分拣箱学过，新来一个竖直放置的快递时失败
   - 判断指标：物体角度 / 物体类型 / 堆积方式

2. **遮挡/视角切换** — 看不全或角度极端
   - 例：抓地上的零件时，自己的手臂挡住了视线
   - 判断指标：遮挡率 / 视线角度 / 物体深度

3. **极端光照** — 逆光 / 闪光 / 阴影
   - 例：上午太阳直射，图像过曝；下午光线柔和，模型就能抓
   - 判断指标：图像亮度直方图 / 对比度 / 色温

4. **高精度任务** — 3mm 级别精度
   - 例：VLA 是数据驱动，没有 IK 的几何可靠性
   - 判断指标：任务精度要求 / 物体尺寸

5. **动态环境** — 背景/干扰物体在移动
   - 例：传送带上的其他物体 / 人走近 / 周边机械动作
   - 判断指标：场景复杂度 / 动作数量 / 背景熵

## 第 17 天 - 根因分析框架（场景版）

### 核心问题

> **什么时候该用 VLA，什么时候该用传统方案？**

不是「VLA 一定更好」，而是**权衡矩阵**。

### 关键指标

| 指标 | VLA 友好 | 传统友好 | 原因 |
|:---|:---|:---|:---|
| **品种数** | > 50 | < 10 | VLA 泛化能力 vs 脚本工作量 |
| **节拍** | 任意 | < 500ms | VLA 推理延迟通常 0.5-2s |
| **精度** | < 2cm | < 1mm | VLA 是数据驱动，IK/MoveIt 是几何精确 |
| **光照变化** | 大 | 小 | VLA 自适应，视觉系统怕光 |
| **物体变异** | 大 | 小 | VLA 学过才能泛化 |
| **冷启动数据** | > 1000 轨迹 | 0 | VLA 需要数据飞轮 |
| **成本容忍** | 成功率 > 80% | 成功率 > 99% | VLA 不完美，需要兜底 |

## 第 17 天 - 根因分析框架

### 场景

产线成功率从 99% 突然降到 92%。怎么排查？

### 五层诊断流程

```
┌─────────────────────┐
│   成功率下降了       │
└──────────┬──────────┘
           │
    ┌──────▼──────┐
    │ 1. 是新品吗？  │  → 来料变了？拆包方式？堆积方式？
    └──────┬──────┘
           │
    ┌──────▼────────────┐
    │ 2. 图像质量变了吗？ │  → 相机松了？镜头脏了？光线变了？
    └──────┬────────────┘
           │
    ┌──────▼──────────────┐
    │ 3. 模型推理错了吗？   │  → 对比输入输出，看中间层激活值
    └──────┬──────────────┘
           │
    ┌──────▼──────────────┐
    │ 4. 规划失败了吗？     │  → 抓取点冲突？运动不可达？
    └──────┬──────────────┘
           │
    ┌──────▼─────────────┐
    │ 5. 执行精度失了吗？  │  → 机械臂精度漂移？末端执行器磨损？
    └─────────────────────┘
```

### 诊断表格

| 层级 | 信号 | 检查方法 | 兜底手段 |
|:---|:---|:---|:---|
| **来料** | 新品/新堆积 | 比对历史图片 | 重新标注 / 启动迁移学习 |
| **图像** | 像素均值下移 / 对比度低 | 直方图分析 + 图像质量打分 | 重新校准光照 / 更换相机参数 |
| **推理** | 置信度下降 | 对比历史推理置信度分布 | 快速微调（LoRA） / 切换备用模型 |
| **规划** | 可达性警告 | 检查运动规划报错 | 调整抓取点 / 调整机械臂位置 |
| **执行** | 位置误差 > 阈值 | 运动结束后对比期望位置 | 标定仪器 / 维修机械臂 |

### 产出要求

- [ ] 制作一个 **诊断决策树**（图表 or 流程图）
- [ ] 每一层附上 **具体的检查命令 / 代码**
  - 例：图像质量检查 → `cv2.calcHist()` 计算亮度分布
  - 例：推理置信度 → 导出模型输出的 softmax 分布
- [ ] 写一个 **假设故事**：某个产线今天成功率掉了，你怎么排查？5 分钟内给出初步诊断。

### 场景差异诊断

```
成功率下降了！不同场景的排查优先级完全不同：

场景A（物流）：来料变化 → 图像质量 → 模型推理 → 规划 → 执行
  因为品种数大类繁多，"来料变了"概率最高

场景B（制造）：执行精度 → 模型推理 → 图像质量 → 规划 → 来料
  因为"精度是生命线"，先查执行层漂移

场景C（农业）：图像质量 → 传感器状态 → 模型推理 → 来料 → 规划
  因为室外环境变化最大，光照/传感器老化最可能是根因
```

### 场景增强任务

学完诊断流程后，**模拟三个场景的诊断**：
1. **场景A**：某天上午 10 点，仓储机器人的抓取成功率从 98% 掉到 85%。你看了时间是上午 10 点。第一反应查什么？
2. **场景B**：产线 VLA 系统精度从 99.5% 掉到 97%。项目经理催你 1 小时内给出诊断。你的排查顺序是什么？
3. **场景C**：农业机器人在阴天和晴天的表现差距很大。是模型问题还是传感器问题？怎么验证？
