## 1. 核心认知：为什么 VLA 都是"双系统"

人类神经系统 = **有意识思考**（慢，要动脑）+ **反射**（快，不过脑子）

所有 2026 年的 VLA 模型都学了这一套：

```
慢系统（System 2）           快系统（System 1）          反射层（System 0）
  │                              │                          │
  │  ~7-10 Hz                    │  ~30-120 Hz               │  ~1 kHz
  │  理解场景                     │  生成动作                   │  紧急反应
  │  "桌上的杯子要在右边拿"        │  关节角序列                  │  碰撞瞬间停
  │                              │                          │
  ▼                              ▼                          ▼
┌──────────┐               ┌──────────┐               ┌──────────┐
│ VLM 视觉  │               │ DiT 扩散  │               │ 轻量网络  │
│ 语言模型  │  ──文本指令──▶ │ Transformer│ ──关节指令──▶ │ 电机控制  │
│ ~2B 参数  │               │ ~32 层    │               │ ~10M 参数 │
└──────────┘               └──────────┘               └──────────┘
```

> 类比你的 Agent 产品：慢系统 = LLM 做意图理解；快系统 = 工作流编排保证确定性输出。架构思路完全一样。

## 3. 关键区别：Action CoT vs 直接输出

这是 GO-2 最独特的设计：

```
其他 VLA：                         GO-2：
 图片 → 模型 → 关节角                图片 → 模型 → 动作步骤 → 关节角
                                              │
                                              ├ "靠近杯子"
                                              ├ "对准把手"       ← 人类可读！
                                              ├ "合拢手指"
                                              └ "上抬手臂"
```

**好处**：中间步骤是人类可理解的，出错了能知道是哪一步错了。类比你的 Agent 产品——工作流步骤可追溯 > LLM 黑盒输出。

## 5. 面试时怎么讲

> "现在的 VLA 模型都是大小脑双系统架构。NVIDIA GR00T 和智元 GO-2 设计思路类似——慢系统做理解规划，快系统做实时动作。GO-2 的差异在于加了动作思维链，中间步骤可解释。这跟我在歌尔做 Agent 时的大模型 + 工作流兜底是一个逻辑——慢系统负责泛化，确定性脚本负责兜底。"
