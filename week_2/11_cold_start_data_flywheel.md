# Day 11 · 冷启动 + 数据飞轮

> 具身智能方案岗最核心的业务判断
> 🕐 预计用时：45 min

---

## 1. 问题定义

```
场景：智元中标了一个 3C 产线上下料项目
挑战：零产线数据，3 个月内要达到 99% 成功率
问题：先采什么数据？采多少？怎么设计数据飞轮？
```

---

## 2. 冷启动三阶段

这是工业落地的标准套路，跟你 Agent 产品的 MVP 策略一模一样：

```
第1阶段（第1-2周）：规则兜底 + 仿真预训练
  ├─ 上仿真（Genie Sim 3.0）生成 1000 条合成数据
  ├─ 真机上跑确定性脚本（MoveIt 固定轨迹）
  ├─ 目标：能跑起来，成功率 ~80%
  └─ 价值：产线不停机，同时积累真机数据

第2阶段（第3-8周）：遥操作 + 模型迭代
  ├─ 工人遥控操作 50-100 次演示
  ├─ 每次演示自动录制（图片+关节角）→ 训练数据
  ├─ 每周微调一次模型
  ├─ 目标：成功率 80% → 95%
  └─ 价值：模型开始具备泛化能力

第3阶段（第9-12周）：飞轮自转
  ├─ 产线自动运行中，筛选「模型不确定的帧」→ 人工标注
  ├─ 模型自信心高的帧 → 直接执行
  ├─ 标注数据喂回训练 → 下周成功率 +1%
  ├─ 目标：成功率 95% → 99%+
  └─ 价值：飞轮转起来，人工介入越来越少
```

---

## 3. 什么数据该喂给模型，什么不该

```
✅ 该喂的数据：
  - 新 SKU 的遥操作演示
  - 模型低置信度的执行记录（模型自己说"我不确定"）
  - 客户反馈的失败案例（有根因标签的）
  - 换了来料批次后的运行数据

❌ 不该喂的数据：
  - 相机脏了导致的模糊图片（不是模型问题）
  - 机械故障导致的失败（夹具松了，跟模型无关）
  - 人工操作失误的演示
  - 同一场景的 1000 次重复（边际收益递减）

→ 这跟 CV 里清洗训练数据一样——不是数据越多越好，是质量越高越好。
```

---

## 4. 跟 CV 数据飞轮的类比

```
CV 的数据飞轮（你在歌尔做的）：
  产线摄像头 → 模型推理 → 低置信度样本 → 人工复核/标注
  → 加入训练集 → 模型更新 → 下周漏检率下降

机器人数据飞轮：
  VLA 推理 → 模型不确定的动作 → 人工遥操作/标注
  → 加入训练集 → 模型更新 → 下周成功率上升

同一个闭环！区别只是"标注"的方式：
  CV：人画框
  机器人：人遥控做一遍（做完自动录好了）
```

---

## 5. 面试怎么讲

> "具身智能的落地冷启动分三阶段：先用仿真预训练 + 确定性兜底保证产线不停，再用遥操作数据微调提升泛化，最后飞轮自转——产线运行数据自动筛选、自动标注、自动回训。我在歌尔做视觉模型时走的也是这个飞轮逻辑——低置信度样本回传标注、重训、上线。闭环是一样的，只是数据从图片变成了动作序列。"
