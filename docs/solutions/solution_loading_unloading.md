## 一、场景定义

### 工位描述

```
流水线来料 → 机械臂抓取 → 放到测试治具 → 测试完成 → 机械臂取回 → 分拣良品/不良品
```

### 关键参数（典型 3C）

| 参数 | 值 | 说明 |
|:---|:---|:---|
| 节拍 | ≤ 8s / 件 | 不能更慢 |
| 精度 | ±0.5 mm | 治具定位要求 |
| 品种 | 50+ SKU / 天 | 频繁换产 |
| 来料一致性 | ±3 mm | 传送带定位精度 |

### 传统方案痛点

```
传统方案 = 固定夹具 + 示教器编程 + 视觉引导
  ├─ 换产时重新示教 → 停机 1-2 小时
  ├─ 来料偏差大时视觉抓不到 → 需要人工调参
  └─ 多品种管理复杂 → 程序多到没人记得住

→ 工厂需要的不是「更精确」，而是「更省人」
```

### 第二层：视觉定位重试（**你的老本行**）

> ⚠️ **精度澄清**：单张远距离 RGB 图像到不了 mm 级。工业 mm 级精度靠的是 **Eye-in-Hand（手腕相机）+ 两步走策略**。

#### 两步定位流程

```
第一步：粗定位（Eye-to-Hand 外部相机）
  ┌─────────────────────────────────────┐
  │  头顶固定相机 / 传送带传感器          │
  │  分辨率: 1280×1024                    │
  │  → 1 像素 ≈ 2-5 mm                   │
  │  → 精度 ±10-20 mm（足够告诉"在哪个区"） │
  │  → 输出: 工件的粗略位置                │
  └──────────────────┬──────────────────┘
                     │
                     ▼
第二步：精定位（Eye-in-Hand 手腕相机）
  ┌─────────────────────────────────────┐
  │  机械臂移动到工件上方 20-30cm         │
  │  手腕上的相机近距离拍照               │
  │  → 1 像素 ≈ 0.05-0.1 mm             │
  │  → 精度 ±0.3-0.5 mm ✅               │
  │  → 配合手眼标定 + IK 求解精确抓取      │
  └─────────────────────────────────────┘
```

**为什么能到 mm 级？不是一张图厉害，是「走近了再拍」厉害：**

```
距离 1m 时:  1 像素 ≈ 3 mm    → 你质疑得对，确实不行
距离 20cm 时: 1 像素 ≈ 0.06 mm → 加上标定，±0.3mm 完全可行

工业上就是这么干的——没有哪个产线用 1 米外的相机做精定位。
```

| 技术点 | 你需要知道什么 | 学习来源 |
|:---|:---|:---|
| 工业相机选型 | 彩色/黑白/3D，视野/分辨率怎么算 | 你的歌尔经验 ✅ |
| YOLO 目标检测（粗定位） | 检测工件 → 粗定位发送给机械臂 | 你的歌尔经验 ✅ |
| Eye-in-Hand 精定位 | 手腕相机靠近后精确拍照 | 补一下原理即可 |
| 手眼标定 | 相机→机械臂坐标变换 | 补一下原理即可 |
| IK 求解 | 位姿 → 关节角 → 执行 | 已学的 Day 5 |

**面试话术锚点：**
> "VLA 泛化失败后，切到传统视觉做精确纠偏。第一步用头顶相机做粗定位——YOLO 检测工件位置，告诉机械臂'往那个区域走'。第二步关键——机械臂移动到工件上方 20cm 处，手腕上的 Eye-in-Hand 相机近距离拍照，这时 1 像素只有 0.06mm，加上标定，精度 ±0.3mm，完全满足 3C 上下料的 ±0.5mm 要求。我在歌尔用工业视觉做了三年，这一层我最熟。最多重试 2 次，节拍不允许无限循环。"

### 第三层：力控兜底

| 技术点 | 你需要知道什么 | 学习来源 |

**面试话术锚点：**
> "视觉也找不到工件时，切到力控兜底——机械臂慢速逼近大概位置，碰到东西就合拢。不用看到，碰到就行。这跟我之前在越南贴胶带一个思路——物理手段替代算法精度。成本是每个工位多 1-2 秒，但避免了一次停机 10 分钟。"

### 第四层：人工介入

| 技术点 | 你需要知道什么 | 学习来源 |
|:---|:---|:---|
| MES 对接 | 报警信号怎么发，产线怎么不停 | 你的歌尔经验 ✅ |
| 异常处理流程 | 机械臂归位 → 等待 → 恢复 | 你的 PMP 经验 ✅ |

**面试话术锚点：**
> "我设计系统的原则是：不要让任何一个单点故障导致产线停线。前三层全失败后，机械臂自动归位，MES 报警，其他工位继续跑，人工 30 秒内过来处理。这跟我做 Agent 产品的设计逻辑一样——大模型做泛化，工作流做兜底，人工做最后一道关。"
