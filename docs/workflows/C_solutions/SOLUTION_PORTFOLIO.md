## 🏭 场景 A：仓储物流

> **对标：** Amazon Vulcan + Agility Digit
> **你的缝合经验：** AGV + 多相机仓储盘点

### 核心逻辑

| 维度 | 说明 |
|:---|:---|
| **为什么 VLA 行** | SKU 多需泛化（VLA 擅长）、精度 ±1cm 就够（VLA 能达到）、环境相对结构化 |
| **为什么传统不行** | 换型成本高，每换一个 SKU 要重新编程 |
| **冷启动** | 标准品建基线 → 非标打溢价 → 数据飞轮扩展 |
| **ROI 要点** | 替代 3 名操作工/台，年省 ¥43.2万人工 + ¥9万良率 |

### 关键数字
- 典型仓库 SKU：500-2000+
- G2 定位精度：±1-5cm（满足仓储需求）
- AGV + 多相机方案：先覆盖 60% 高频货位

## 🏭 专案：3C 上下料工位（最深的一篇）

> **这是你的敲门砖——你最熟的场景，方案最细。**

### 工位描述
```
流水线来料 → 机械臂抓取 → 放到测试治具 → 测试完成 → 机械臂取回 → 分拣良品/不良品
```

### 方案亮点

| 模块 | 说明 |
|:---|:---|
| **视觉感知** | YOLO 检测工件位置 → 计算抓取位姿（AI Camera 经验） |
| **运动规划** | MoveIt2 + ROS2 → Panda 执行抓取 |
| **VLA 推理** | 云端 OpenPI / GR00T → 泛化到新工件 |
| **四级兜底** | VLA 失败 → 视觉重试（重新定位）→ 力控调整（力矩传感器）→ 报警人工 |
| **MES 集成** | 良品/不良品分拣 + 数据上报 |

### 你的经验缝合
- AI Camera（视觉感知）→ 工件检测 + 位姿估计
- 歌尔产线经验 → 节拍、良率、换型时间
- Agent 产品经验 → 兜底编排 + 异常处理

> 完整方案 → `solution_loading_unloading.md`

## 📐 场景决策树：VLA vs 传统 vs 混合

```
客户需求
├── 高速产线（节拍 <5s）、完全固定、±0.1mm → **传统方案**
│     机器视觉+PLC，生命周期成本最低
│
├── 多型号混产、SKU 多、±1-5mm 精度够 → **VLA 方案**
│     龙旗 IRR 78.2%，远超制造业 15% 基准线
│
└── 两者兼有（最常见）→ **混合方案**
       固定动作 → 规则控制（省算力）
       高精度动作 → RL 专项优化（提良率）
       通用操作 → VLA 端到端（省换型时间）
```
