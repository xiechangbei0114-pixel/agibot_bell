## ① 机器人分类（工业 vs 协作 vs 人形 vs AGV）

| 类型 | 自由度 | 精度 | 典型应用 | 面试锚点 |
|:---|:---:|:---:|:---|---:|
| 工业臂 | 4-6 | ±0.02mm | 焊接、喷涂 | 歌尔的自动化产线 |
| 协作臂 | 6-7 | ±0.1mm | 上下料、装配 | Panda（你用的仿真）|
| 人形 | 26-40+ | ±1-5mm | 通用操作 | 智元 G2（26 DOF）|
| AGV/AMR | 2-3 | ±10mm | 搬运、盘点 | 你的 AGV 仓储项目 |

**面试话术：** "形态服务场景——固定工位高精度选工业臂，多型号混产选协作臂，通用场景看人形。G2 轮式半人形在工业场景是当前 ROI 最优解。"

## ② 四层控制架构

```
感知层 (相机/力传感器) → 决策层 (VLA/运动规划) → 执行层 (关节电机) → 通信层 (ROS2 Topic)
```

**产线映射：**
| 你的 AGV 项目 | 机械臂场景 |
|:---|---:|
| 相机拍货架 → YOLO 检测 | 相机拍工件 → VLA 推理 |
| 路径规划 → AGV 移动 | 运动规划 → 关节执行 |
| MES 上报 → 闭环 | MES 上报 → 闭环 |

**面试话术：** "核心是感知→决策→执行→通信的闭环。我做了 AGV 盘点，架构是一样的。"

---

## ③ ROS3 通信 + AimRT

| 方式 | 用途 | 类比你的经验 |
|:---|:---|---:|
| Topic（发布/订阅） | 相机图像流 | 类似 MQTT 消息 |
| Service（请求/响应） | 触发拍照 | 类似 HTTP API |
| Action（目标/反馈/结果） | 机械臂抓取 | 适时长任务 |

**AimRT（智元下一代通信框架）：** 比 ROS2 更低延迟、更确定性的工业实时通信。

## ④ FK / IK（正/逆运动学）

| | 输入 → 输出 | 场景 |
|:---|---:|:---:|
| FK | 关节角 → 末端位姿 | 仿真验证当前状态 |
| IK | 末端位姿 → 关节角 | **抓取时用——你说"抓这里"，IK算出关节怎么转** |

**面试话术：** "IK 解决的是——视觉告诉我要抓哪个位置，机械臂怎么转关节才能到那里。MoveIt2 里你传 PoseStamped，它内部就解了 IK。"

## ⑤ VLA 大小脑架构

```
相机图像
    │
    ▼
┌──────────┐
│   大脑    │ ← 多模态大模型（VLM）理解场景语义
│  (VLM)   │    输出：语言指令/目标物体
└────┬─────┘
     │
     ▼
┌──────────┐
│   小脑    │ ← 动作专家模型（Diffusion Policy）
│  (Policy) │    输出：关节动作序列
└────┬─────┘
     │
     ▼
   机械臂执行
```

**智元 GO-2：** ViLLA 架构（IROS 2025），ACoT 在动作空间推理（CVPR 2026），LIBERO 98.5% SOTA。

## ⑥ 三大模型对比（VLA / RL / IL）

| | VLA | RL（强化学习） | IL（模仿学习） |
|:---|:---|:---:|:---:|
| 数据来源 | 互联网图文+动作 | 试错（reward） | 人类示教 |
| 泛化性 | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ |
| 精度 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 产线场景 | 多型号混产 | 高精度重复 | 固定动作 |

**面试话术：** "固定动作用 RL 专项优化提良率，通用操作靠 VLA 省换型时间，混合方案用最低总持有成本覆盖最广作业范围。"

> **龙旗张总原话印证：** "我们并不是所有工作内容都是用的真机强化学习...也针对于他的动作做了行为素的一个优化。"
> → 说明产线上是混合方案，不是全 VLA

## ⑧ RL vs IL

| | RL | IL |
|:---|:---|:---:|
| 要什么 | Reward 函数 + 大量交互 | 人类示教数据 |
| 适合 | 高精度、可模拟场景 | 复杂操作（遥操作录数据）|
| 缺点 | 训练慢、Sim2Real 难 | 学不到比示教更好的策略 |

**产线判断：** "RL 适合精度要求高的固定动作，IL 适合人容易示教的复杂操作。智元龙旗的策略是 RL 提良率 + VLA 做泛化。"

## ⑨ Sim2Real

**核心挑战：** 仿真训练的模型到真机有性能下降。

**智元的解法（龙旗案例）：**
1. 仿真数字孪生（90% 研发在 Genie Sim）
2. 副产线调试（1 个月适应现场光照/布局）
3. 主产线闲时压测（8h+ 连续运行）
4. 正式并线，MTBF 从几小时提升到数百小时
5. 新技能 1h 学到 99%+

> **姚院原话：** "最近具身智能圈大家说的一个概念叫做 beyond algorithms...真正要把一套机器人稳定的部署在工业产线，involve的东西是非常多的——首先就是一个稳定的硬件，其次精密标定，它不是单单一个模型就可以百分之百解决所有问题。"
4. Sim-to-Real gap < 10%（Genie Sim 基准）

## ⑩ 冷启动三阶段

```
规则驱动 → 遥操作采集 → 数据飞轮
   ↓            ↓            ↓
先跑通流程   录示教数据   模型替代规则
```

**智元龙旗路径：**
| 阶段 | 时间 | 内容 |
|:---:|:---:|:---|
| 仿真 | ~4.5月（并行） | Genie Sim 数字孪生 |
| 副产线 | ~1 月 | 适应现场光照/布局 |
| 主产线压测 | 8h+ | 利用闲时，至少8小时连续稳定运行 |
| 正式并线 | 2026.03.16 | 总周期 ~6 个月 |

> **姚院原话——冷启动分步法：** "真正上产线要分步骤的：首先在自己公司实验室搭建模拟环境，包括仿真的、真实的；到一定成熟度之后进入工厂，先从副产线开始，去适应现场光照环境；并线前一周利用主产线闲时做连续压测，保证至少8小时稳定运行、不会停线，达到连续压测通过的标准之后，它就进入产线。"

## ⑪ 兜底金字塔（你的核心武器）

```
         ┌──────────┐
         │ ④ 报警   │ ← 人工介入（最终保障）
         ├──────────┤
         │ ③ 力控   │ ← 力矩传感器调整
         ├──────────┤
         │ ② 视觉   │ ← 失败重新定位
         ├──────────┤
         │ ① VLA    │ ← 端到端推理（先试）
         └──────────┘
```

**面试话术：** "VLA 失败率不是 0，但四级兜底可让产线直通率到 99.91%。每往下一级增加可靠性，但降低效率——这是可靠性-效率 trade-off 的工程决策。"

## ⑫ 范式四阶段演进

```
Era 1: 规则驱动       — 示教器/PLC，精度高换型慢
Era 2: 模仿学习       — 视觉感知+运动规划，你的AI Camera在这
Era 3: VLA 端到端     — 图像直接出动作，当前主流
Era 4: 世界模型+Agent — 智元 GO-2 + Genie Studio Agent
```

**产线不是替代关系：** 龙旗四代技术混合使用——固定动作用 Era 1 省算力，高精度用 Era 2 提良率，通用操作靠 Era 3 省换型。
