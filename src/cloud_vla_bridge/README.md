# ☁️ 云端 VLA → 本地 Panda 仿真

> 架构：AutoDL 云端跑 VLA 推理 → 网络传输 → WSL2 ROS2 控制 Panda

---

## 一、整体架构

```
                      ☁️ 云端 (AutoDL / 阿里云)
┌─────────────────────────────────────────────────────┐
│                                                     │
│  相机画面 ──→ VLA 模型 (Pi0 / OpenVLA / GO-2 API)   │
│                    │                                │
│                    ▼                                │
│              输出: 末端位姿(xyz+四元数)               │
│              或:   关节角 [j1...j7]                  │
│                    │                                │
│                    ▼                                │
│              WebSocket / gRPC 服务端                  │
└────────────────────┬────────────────────────────────┘
                     │ 网络 (局域网/公网)
                     ▼
                  💻 你的电脑 (WSL2)
┌─────────────────────────────────────────────────────┐
│  WebSocket / gRPC 客户端                             │
│         │                                           │
│         ▼                                           │
│  ROS2 Node (bridge_node)                            │
│         │                                           │
│         ├── 收到位姿 → 发布到 /vla_pose              │
│         │                                           │
│         ▼                                           │
│  Panda MoveIt2 执行（Gazebo 仿真）                   │
│                                                     │
│  同时订阅 /parts_detected（来料信号）                 │
│  触发 → 请求云端 VLA 推理                            │
└─────────────────────────────────────────────────────┘
```

---

## 二、你需要什么

### 云端：AutoDL 租 GPU（¥2-5/小时）

| 配置 | 价格 | 能跑什么 |
|:---|:---:|:---|
| RTX 4090 24GB | ~¥4/小时 | Pi0.5 / OpenVLA 7B |
| RTX 3090 24GB | ~¥2/小时 | Pi0 / 小模型 |
| RTX 4080 16GB | ~¥2.5/小时 | 小模型推理 |

> 注意：GO-2 是智元闭源模型，你拿不到权重。但你可以用开源的 Pi0 / Pi0.5 或 OpenVLA 做替代。

### 本地：你已有的

- WSL2 + Ubuntu 22.04 ✅
- ROS2 Humble ✅
- Panda + MoveIt2 + Gazebo ✅
- `panda_move.py`（控制脚本）✅
- `agibot_demo.py`（产线通信 Demo）✅

## 三、通信方式对比

| 方式 | 延迟 | 难度 | 推荐 |
|:---|:---:|:---:|:---:|
| **WebSocket** | ~5-20ms | ⭐⭐ | ✅ **推荐**——简单通用 |
| ROS2 Topic 跨网络 | ~2-10ms | ⭐⭐⭐⭐ | ROS2 原生方案，需 DDS 配置 |
| gRPC | ~3-15ms | ⭐⭐⭐ | 高性能，配置稍复杂 |
| HTTP REST | ~50-200ms | ⭐ | 简单但延迟高，不适合实时 |

## 四、简化版：不需要云端也能跑

如果不想租云 GPU，可以用**一个 Python 脚本来模拟 VLA 输出**：

```
你的电脑
┌────────────────────────────────────────┐
│  simulate_vla.py                       │
│  ┌────────────────────────────────┐    │
│  │ 收到 /parts_detected（来料）    │    │
│  │ → 随机生成一个"VLA 推理结果"    │    │
│  │ → 发布到 /vla_pose             │    │
│  └──────────┬─────────────────────┘    │
│             ▼                          │
│  ROS2 + Panda MoveIt2                  │
│  → 收到位姿 → 执行抓取                  │
└────────────────────────────────────────┘
```

这就是你的 **「VLA 推理 + 执行」Demo**——云端 GPU 换成本地脚本模拟，但**架构一模一样**。
