## 1. FK vs IK —— 核心直觉

> 你用**自己的身体**就能理解：闭眼伸出右手——你知道肩膀和肘子弯了多少度（关节角），你能算出指尖在哪（FK）。但如果我说"去碰咖啡杯"，你要反过来推算关节该怎么弯（IK）。

| | FK（Forward Kinematics） | IK（Inverse Kinematics） |
|:---|:---|:---|
| **问题** | 关节点角度 → 末端在哪？ | 末端目标位置 → 关节该转多少度？ |
| **公式方向** | θ₁,θ₂,θ₃... → (x, y, z, roll, pitch, yaw) | (x, y, z, roll, pitch, yaw) → θ₁,θ₂,θ₃... |
| **解的数量** | 1 个（唯一） | 0、1、多个、∞（无穷） |
| **计算难度** | 简单三角函数 | 非线性方程组，需数值优化 |
| **日常类比** | 「弯了手臂，手在哪？」 | 「看到杯子，手臂怎么够到？」 |

---

## 2. 为什么 IK 难？

以最简单的**2 连杆机械臂**为例：

### FK（顺手推）
```
x = l₁·cos(α) + l₂·cos(α+β)
y = l₁·sin(α) + l₂·sin(α+β)
```
代入 α=45°, β=30° → 立即得到 (x, y) = (0.65, 1.10)。唯一解。

### IK（逆向推）
已知目标位置 (x, y) = (0.8, 0.6)，求 α 和 β。

这叫"非线性求逆"，会遇到四个问题：

| 问题 | 说明 | 实例 |
|:---|:---|:---|
| **多解** | 肘关节朝上还是朝下？都能到 | 你的手臂可以用两种姿势够到脸 |
| **无解** | 目标太远，够不到 | 想要抓 2 米外的物体 |
| **奇异性** | 某些角度组合让自由度降级 | 手臂绷直时无法再往前伸 |
| **冗余** | 7-DOF 臂去够 3D 点 → ∞ 个解 | 人形手臂比目标维度多 |

**实际做法**：用 Jacobian 矩阵迭代数值求解（如阻尼最小二乘法 DLS），MoveIt2 内置的 IK 求解器（KDL/Trac-IK/BioIK）就是干这个的。

## 3. 你在 Panda 上能看到 FK/IK

启动 Panda 仿真后，你在 rviz 里**拖拽末端标记球**就是**调 IK**——MoveIt2 背后的 IK 求解器在实时算「要实现这个末端位姿，关节角应该是多少」。

在 Python 代码里也是这样：
```python
# 给定目标位姿 (x, y, z) → IK 求解器算出关节角 → 规划轨迹 → 执行
```

## 5. 面试自检答案

> **Q: 机械臂要抓桌上零件，IK 起什么作用？为什么不用 FK？**

**A:**
- **IK 的作用**：我知道零件的位姿（来自视觉），需要反算出 6 个关节各转多少度才能让夹爪到达那个位姿。IK 就是做这个"反算"的。
- **为什么不用 FK**：FK 是正着算的——「给我角度，告诉你末端在哪」。抓取任务的输入是**目标末端位姿**，输出是**关节角度**，方向反了。FK 可以用来验证「这个关节角对不对」，但不能用来做规划。

> 💡 **加分回答**：在实际系统中，MoveIt2 的 Motion Planning Pipeline 先用 IK 求解器算出目标关节角，再用 RRT 等运动规划器生成从当前角到目标角的无碰撞轨迹，最后发给 ros2_control 执行。
