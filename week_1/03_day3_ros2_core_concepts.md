# Day 3 · ROS2 核心概念（06.19）

## 学习目标

搞清 Node / Topic / Service / Action 四个概念的区别和适用场景。

## 知识框架

| 机制 | 通信模式 | 类比 | 适用场景 |
|:---|:---|:---|:---|
| **Topic** | 发布/订阅（1→N） | 广播电台 | 传感器数据流（相机、激光）、状态持续广播 |
| **Service** | 请求/响应（1→1） | HTTP API | 一次性查询（「给我现在的位置」）、参数设置 |
| **Action** | 带反馈的异步任务（1→1） | 外卖下单→接单→配送→完成 | **抓取、导航**等长耗时任务，需要进度反馈和可取消 |

```
Node = 进程（执行单元）
Topic = 数据管道（传什么）
Service = 同步问答（问什么）
Action = 异步任务（做什么+进度+可取消）
```
