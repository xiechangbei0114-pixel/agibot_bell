## 3. 通关自检（三条，必须全过）

- [ ] **① 脱稿讲 5 分钟「从传感器到执行器的完整数据链路」**

  提纲：
  ```
  开头（30s）：机械臂控制分四层——感知、规划、控制、执行。和我做过的
              AGV 一样，只是维度不同。

  感知层（60s）：相机拍点云，力传感器测接触力，编码器读关节角度。
               类比 AGV 的激光雷达——都是「把物理世界变成数据」。

  规划层（90s）：MoveIt2 用 IK 把目标位姿反算成关节角，再用 RRT
               在 6D 空间搜索无碰撞轨迹。AGV 用的是 A* 在 2D 搜
               索，本质都是搜索算法，维度爆了所以要换 RRT。

  控制层（60s）：轨迹出来了要变成电机指令——PID 闭环跟踪，每毫秒
               比较「该在哪」和「实际在哪」然后修正。AGV 是速度 PID，
               机械臂是关节力矩 PID，控制逻辑一样。

  执行层（30s）：电机转起来，通过减速器增扭，关节运动，夹爪到目标。

  总结（30s）：四层架构是通用的，我从小车升维到机械臂，
             不是换赛道，是同一个框架下的维度扩展。
  ```

## 4. 本周面试话术汇总

### 30 秒自我介绍（随时能讲）
```
我做视觉和 Agent 出身，最近在学习具身智能机器人。
我在歌尔做过 AGV 调度和产线 MES 对接，发现机器人控制架构和
我之前做的其实是通的——AGV 是 2D 的机器人，机械臂是 6D 的机器人。
我搭了一套 ROS2 + MoveIt2 环境，能用 Python 控制 Panda 机械臂运动。
```

### 2 分钟缝合话术（AGV → 机械臂）
```
我在歌尔做 AGV 的视觉盘点，当时调度系统包括感知→规划→控制→执行四个环节。
现在我学机械臂发现是同一个框架。AGV 用 A* 做路径规划，机械臂用 RRT——算法换
了是因为维度从 3DOF 升到 6DOF，但搜索最优路径的思想一样。AGV 用 Topic 广播
位置给 MES，机械臂用 Action 做抓取——因为抓取耗时且需要结果确认，而状态广播
不需要。所以我的经验迁移不是「从零开始」，而是「从 2D 升维到 6D」。
```
