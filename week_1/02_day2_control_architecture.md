## 1. 经典三层架构

大多数现代机器人系统采用**感知→规划→控制→执行**的流水线架构。教科书《Introduction to Autonomous Robots》将这一架构视为机器人控制的基本范式。

```
┌────────────────────────────────────────┐
│           1. 感知层 (PERCEPTION)        │
│  ┌──────────┐ ┌──────────┐ ┌────────┐ │
│  │ 相机点云  │ │ 力矩传感  │ │ 编码器  │ │
│  │ 物体检测  │ │ 力觉数据  │ │ 关节位  │ │
│  │ 位姿估计  │ │ 碰撞检测  │ │ 速度值  │ │
│  └──────────┘ └──────────┘ └────────┘ │
│         输出: 结构化环境模型              │
├────────────────────────────────────────┤
│           2. 规划层 (PLANNING)          │
│  ┌──────────────┐ ┌──────────────────┐ │
│  │ 任务规划      │ │ 运动规划 (Motion) │ │
│  │ Task Level:   │ │ Path/Trajectory  │ │
│  │ 拆成子任务    │ │ A*/RRT 路径搜索  │ │
│  └──────────────┘ └──────────────────┘ │
│         输出: 无碰撞轨迹                 │
├────────────────────────────────────────┤
│           3. 控制层 (CONTROL)           │
│  ┌──────────┐ ┌──────────┐ ┌────────┐ │
│  │ 轨迹跟踪  │ │ 力位混合  │ │ 伺服控制 │
│  │ PID/MPC  │ │ 阻抗控制  │ │ 电流环   │
│  │ 关节空间  │ │ 笛卡尔   │ │ 速度环   │
│  └──────────┘ └──────────┘ └────────┘ │
│         输出: 关节力矩/速度指令          │
├────────────────────────────────────────┤
│           4. 执行层 (EXECUTION)         │
│  ┌──────────┐ ┌──────────┐ ┌────────┐ │
│  │ 电机驱动  │ │ 减速器    │ │ 末端工具 │
│  │ BLDC/步进 │ │ 谐波/RV   │ │ 夹爪/吸  │
│  │ 转矩输出  │ │ 增扭减速  │ │ 焊枪/主轴│
│  └──────────┘ └──────────┘ └────────┘ │
│         输出: 物理运动                    │
└────────────────────────────────────────┘
```

### 各层详细说明

| 层 | 做什么 | 输入 | 输出 | 关键技术 |
|:---|:---|:---|:---|:---|
| **感知** | 获取环境和自身状态 | 相机/力传感器/编码器 | 点云/位姿/力数据 | YOLO, SAM, 点云配准, 传感器融合 |
| **规划** | 算"怎么走/怎么做" | 感知数据 + 任务目标 | 无碰撞轨迹 | A\*, RRT, CHOMP, STOMP, TrajOpt |
| **控制** | 轨迹→电机指令 | 目标轨迹（位姿序列） | 关节力矩/速度指令 | PID, MPC, 阻抗控制, 导纳控制 |
| **执行** | 物理运动 | 控制指令 + 反馈 | 关节转动 / 夹爪动作 | 伺服驱动, 谐波减速器, BLDC 电机 |

## 2. MoveIt2 中的架构映射

你已经在跑的 MoveIt2 是这套架构的**实际实现**：

```
感知层   ← MoveIt2 Planning Scene（维护碰撞环境）
  │          └─ 接收相机点云 → Octomap 更新
  ↓
规划层   ← MoveIt2 Motion Planning Pipeline
  │          └─ OMPL/RRTConnect 规划器 → 生成轨迹
  ↓
控制层   ← ros2_control（JointTrajectoryController）
  │          └─ 接收轨迹 → PID 跟踪 → 关节指令
  ↓
执行层   ← Gazebo / 真实硬件
             └─ 仿真物理 / 真实电机执行
```

## 3. AGV 与机械臂的架构类比

这是你的面试缝合点——用歌尔 AGV 经验讲机械臂架构：

| 架构层 | AGV | 机械臂 | 本质相同点 |
|:---|:---|:---|:---|
| 感知 | 激光雷达 + 相机 → 障碍物检测 | 3D 相机 + 力矩 → 工件定位 | **传感器→结构化数据** |
| 规划 | A\* + DWA → 全局/局部路径 | RRT + MoveIt → 运动轨迹 | **搜索/优化算法** |
| 控制 | PID → 速度指令（v, ω） | PID/阻抗 → 关节力矩（τ₁...τₙ） | **闭环反馈控制** |
| 执行 | 电机驱动轮子转动 | 伺服电机驱动关节 | **底层物理执行** |

**关键差异**：AGV 在 2D 平面规划（x, y, θ），机械臂在 6D 空间（x, y, z, roll, pitch, yaw）→ **维度爆炸** → 算法从 A\* 升到 RRT。

## 4. 面试话术模板

> "我在歌尔做的 AGV 调度系统，控制架构就是感知→规划→控制→执行四层。AGV 用激光雷达做感知、A* 做路径规划、PID 做速度控制、电机驱动执行。机械臂的逻辑完全一样——只是从 2D 升到 6D，规划算法从 A* 变成了 RRT，控制器从速度 PID 变成了关节力矩 PID。架构思想是通的。"

## 5. 自检答案

> **Q: 你做的 AGV 视觉盘点，控制架构和机械臂有什么异同？**

A: 见上表。补充一点——AGV 的"碰撞检测"是在 2D 激光层面的（检测到障碍停车），机械臂的碰撞检测是在 3D 空间（Octomap + 自碰撞检测），但本质都是"保障安全运动"。架构同构，具体实现升维。
