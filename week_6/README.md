## 📋 第 36 天 - 三场景方案自检清单

### 通用自检（10 问）

- [ ] 场景痛点是否具体到"一个工位"？（不要讲"制造业很缺人"，要讲"这个工位每年损失 XXX"）
- [ ] 方案架构是否有层次（感知→决策→执行→兜底）？
- [ ] ROI 算账是否有 3 个以上数据支撑（回本周期、节省金额、良率提升）？
- [ ] 是否说明了"为什么是智元"而不是别的方案？
- [ ] 兜底链是否完整（至少 3 层）？
- [ ] 是否包含安全/合规说明（至少提及标准号）？
- [ ] 是否有一个"竞争对手的弱点"作为对比？
- [ ] 是否有一个"客户可能问的问题"提前回答了？
- [ ] 是否有一个"实施时间线"让客户觉得靠谱？
- [ ] 整个方案能否在 5 分钟内讲完？

### 场景专属自检

**场景A 仓储物流**：
- [ ] 是否说明 SKU 数量级？（100+）
- [ ] WMS 对接方案是否讲清楚了？
- [ ] VLA 的品种泛化能力如何验证？
- [ ] 兜底的视觉重定向流程画出来了吗？

**场景B 制造业**：
- [ ] 安全标准是否至少列了 3 个（ISO 10218/TS 15066/ISO 91469）？
- [ ] Eye-in-Hand + Eye-to-Hand 混合标定的图画清楚了吗？
- [ ] 五层兜底链每一层是否都有触发条件？
- [ ] MES 对接的接口方式是否有说明？

**场景C 农业/特殊**：
- [ ] RaaS 模式的月付价格是否合理？
- [ ] 买断 vs RaaS vs 人工的 TCO 对比做了吗？
- [ ] 对智元的 3 个启示是否言之有物？
- [ ] 是否说明了"不是让人形机器人去挤奶"？

## 👀 第 37 天 - 找"外部视角"验证

### 方法

**找人评审**（优先级从高到低）：

1. **找在机器人行业的朋友**（最佳）：把方案发过去，问"你客户会问什么问题？"
2. **找同事/非技术朋友**：看他们能不能听懂你的方案
3. **自己查资料对照**：找不到人就用这个方法

### 自己验证的方法

1. 谷歌搜索你方案中的每个关键论断
   - "Amazon Vulcan specifications"
   - "Figure AI BMW deployment results"
   - "Agility Digit ROI"
   - "Lely robot global installations 2026"
2. 对比你的数据是否合理
3. **找到 3 个数据偏差** → 修复

### 修复流程

```
找到漏洞 → 记录在案 → 查资料修正 → 更新到 PPT → 画✓
```

## 🔧 第 38-40 天 - 按场景专项打磨

### 第 38 天：场景B（制造业）专项打磨

**核心任务**：让制造方案从"看起来不错"变成"无懈可击"

| 检查点 | 你的答案 | 是否准备好 |
|:---|:---|:---:|
| ISO 10218 的 3 条核心要求？ | ①安全围栏 ②力/速限制 ③急停 | ⬜ |
| TS 15066 的碰撞力限制是多少？ | < 150N | ⬜ |
| ISO 91469 是什么？ | 人形机器人专项安全标准（2025 草案） | ⬜ |
| Eye-in-Hand vs Eye-to-Hand 优缺点？ | 参见 week_4 方案架构 | ⬜ |
| 五层兜底链逐层触发条件？ | L1 力控→L2 视觉→L3 重试→L4 隔离→L5 人工 | ⬜ |
| 节拍 < 3s 怎么保证？ | VLA 推理可以 pipeline 化 | ⬜ |

**额外练习**：准备回答这个面试题  
> "你们这方案看起来不错。但假如精度从 99.9% 掉到 97%，你怎么排查？"

### 第 39 天：场景A（仓储物流）专项打磨

| 检查点 | 你的答案 | 是否准备好 |

**额外练习**：准备回答这个面试题

## 🏁 第 41 天 - 终版 PPT

### 终版检查清单

- [ ] 所有页字体风格统一
- [ ] 所有图片/图表清晰可读
- [ ] 所有数据来源标注（"来源：IEEE Spectrum 2025" 或 "来源：Amazon Robotics Blog"）
- [ ] 每页文字不超过 50 字（方案岗 PPT 是"留白"艺术）
- [ ] 有一个"口播话术"配套文档（每页怎么讲）
- [ ] 打印成 PDF（面试时带平板或打印件）

### 配套话术文档结构

```
PPT 每页 → 1 分钟话术
├── 场景A 客户 → 翻 P5（场景A方案）+ 讲 3 分钟
├── 场景B 客户 → 翻 P6（场景B方案）+ 讲 3 分钟
├── 场景C 客户 → 翻 P7（场景C方案）+ 讲 3 分钟
└── 通用介绍 → 翻 P3+P4+P8 → 讲 5 分钟
```

## 🎙️ 第 42 天 - 三场景 5 分钟路演录音

### 任务

1. 每个场景录一段 **5 分钟录音**
2. 结构固定（可用计时器辅助）：

```
0:00-0:30  场景痛点（30 秒）
0:30-1:30  方案架构（60 秒）
1:30-2:30  ROI 算账（60 秒）
2:30-3:30  兜底设计（60 秒）
3:30-4:00  为什么是智元（30 秒）
4:00-5:00  客户可能问的问题预回答（60 秒）
```

3. 回听录音，找 **3 个改进点**
4. 修正后再录一遍
5. **第三遍录到满意为止**

### 评估标准

| 维度 | ⭐ 及格 | ⭐⭐ 良好 | ⭐⭐⭐ 优秀 |
|:---|:---|:---|:---|
| 流畅度 | 念稿式 | 偶尔看稿 | 脱稿 |
| 数字准确 | 记不清 | 能说大概 | 脱口而出 3 个精确数字 |
| 技术深度 | 泛泛而谈 | 有 1-2 个技术点 | 有架构/流程/标准细节 |
| 自信度 | 语气不确定 | 语气平稳 | 有对比有判断有态度 |
