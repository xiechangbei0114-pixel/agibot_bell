## 二、选型决策

```
方案中的通信架构（从底到顶）：
┌─────────────────────────────────────┐
│  云端：OPC-UA / MQTT                │  ← 跨系统集成
├─────────────────────────────────────┤
│  边缘网关：OPC-UA → 协议转换        │  ← 中间层
├─────────────────────────────────────┤
│  工控机/PLC：EtherCAT（驱控一体）    │  ← 实时控制
│  或 Profinet（西门子生态）           │
├─────────────────────────────────────┤
│  传感器/执行器：Modbus RTU/Ethernet │  ← 设备层
└─────────────────────────────────────┘
```

**实战判断：**
- **智元龙旗产线**：G2机器人通过 EtherCAT 驱控一体，上层用 OPC-UA 对接龙旗 MES
- **传统3C产线（歌尔）**：设备层走 Modbus，产线PLC走 Profinet，MES对接用 OPC-UA
- **VLA方案**：VLA模型输出→ROS2 Topic→EtherCAT驱控器→关节电机

## 三、面试话术

**Q: "机器人怎么和客户MES对接？"**
> "分两层：底层关节通信走 **EtherCAT**——微秒级周期+纳秒级同步，满足驱控一体的实时性要求。上层与客户MES对接走 **OPC-UA**——标准化信息模型+内置加密，SCADA和ERP都能直接集成，不需要写私有协议。"

**Q: "传统产线都是Modbus/Profinet，你的方案能接吗？"**
> "可以。方案中预留了工业网关做协议转换——Modbus RTU/Profinet → 边缘网关 → OPC-UA → 上层平台。产线现有设备不用动，新增VLA机器人走独立通道，互不干扰。"
