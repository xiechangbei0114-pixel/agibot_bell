# 📊 VLA选型矩阵

> 面试价值：⭐⭐⭐⭐（Q6「VLA vs 传统怎么选？」深度素材）
> 目标：面试官问"什么场景用什么模型"能给出选择框架+推荐

---

## 一、场景化选型决策树

```
客户需求
│
├─ 高速产线（节拍<5s）、完全固定操作、±0.1mm精度
│  → 传统方案（机器视觉+PLC），不需要VLA
│  → 生命周期成本最低，换型成本高但可接受
│
├─ 多型号混产、SKU多、±1-5mm精度够用
│  │
│  ├─ 有算力资源（云GPU/边缘服务器）
│  │  → GO-2 / Pi0.5 / GR00T（云端VLA）
│  │  → 泛化性最好，适合3C换产频繁场景
│  │
│  ├─ 算力受限（低成本边缘设备）
│  │  → VLA推理Lite版本（知识蒸馏+模型量化）
│  │  → 精度略降，但可接受
│  │
│  └─ 精度要求±0.5mm、节拍中等
│     → VLA + Eye-in-Hand精定位（混合方案）✅ 最推荐
│     → VLA粗定位+手腕相机精定位=±0.3mm
│
├─ 高精度重复动作（如螺丝锁付、点胶）
│  → RL 专项优化
│  → 训练慢但跑起来之后精度最高、节拍最快
│
├─ 复杂操作（如线束插拔、活页装配）
│  → IL 模仿学习（人类示教数据）
│  → 人容易示教的操作，RL反而不如IL
│
└─ 新场景、冷启动阶段
   → 规则驱动（先跑通流程）
   → 遥操作采集数据（积累训练数据）
   → 数据飞轮（模型渐进替代规则）
```

## 三、选型决策维度

### 精度 vs 泛化性 trade-off
```
高精度 ←──────────────────→ 高泛化
传统PLC    RL专项    IL模仿    VLA端到端    Agent系统
±0.02mm    ±0.1mm    ±0.5mm    ±1-5mm      ±5-20mm
固定操作   单品种    少品种     多品种       无限品种
（焊点）   （锁付）   （插拔）    （分拣）     （通用）
```

### 算力需求金字塔
```
模型类型          算力需求          延迟           部署位置
Agent系统         ⭐⭐⭐⭐⭐         1-3s          云端
VLA端到端         ⭐⭐⭐⭐          300-500ms     云端/边缘
VLA Lite          ⭐⭐⭐            100-200ms     边缘
视觉模型(YOLO)    ⭐⭐              20-50ms       边缘/嵌入式
传统控制          ⭐                1-10ms        工控机/PLC
```

## 四、面试话术

**Q: "你会给一个3C上下料客户推什么方案？"**
> "我会先问他三个问题来确定选型：第一，一天换多少个品种？第二，上料的精度要求是多少？第三，预算范围？"
>
> 如果结论是"多品种混产±1mm精度"——我推 **VLA混合方案**：
> 1. GO-2做换产泛化（零停机换型）
> 2. 手腕相机Eye-in-Hand做精定位（±0.3mm）
> 3. 力控做兜底（力矩传感器柔性放入）
> 4. 龙旗的数据已经证明这套方案直通率99.91%

**Q: "为什么不用RL训一个端到端？"**
> "RL在固定重复动作上是王者——锁螺丝、点胶，精度高节拍快。但3C产线一天换5个品种、每个品种几千件，RL训一个动作要几百万步交互，换品种全重来。VLA虽然精度不如RL，但换产不用停机。在3C制造这个场景，**泛化带来的省换型时间远大于精度损失的成本**。"

**Q: "GR00T和GO-2你怎么选？"**
> "如果做科研或者快速原型验证，GR00T开源生态好。但如果是要交付给工业客户，GO-2的工业场景数据积累（龙旗6个月冷启动）和Genie Sim的仿真闭环是GR00T没有的。工业客户不看你模型在什么benchmark上多高，他看你能不能稳定7×24运行。**所以选型不是技术选型，是交付能力选型**。"
